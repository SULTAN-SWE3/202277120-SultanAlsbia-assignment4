# AI Usage Report

## Tools Used and Use Cases
- ChatGPT
  - Used to help translate the assignment requirements into a practical implementation plan
  - Used to review front-end logic for filtering, sorting, state management, and validation
  - Used to improve documentation structure and wording
- GitHub Copilot
  - Used for code completion while writing HTML, CSS, and JavaScript
  - Used to speed up repetitive UI and rendering patterns

## Benefits and Challenges
### Benefits
- AI helped reduce setup time by turning the instructions into clear programming tasks.
- It suggested cleaner ways to organize repeated JavaScript logic into small functions.
- It was useful for checking edge cases in validation and API error handling.

### Challenges
- Some AI suggestions were too generic and needed changes to fit the assignment exactly.
- Suggested code sometimes required simplification so the project stayed readable and consistent with a basic portfolio structure.
- API-related suggestions still needed manual checking to make sure the user experience stayed clear when requests failed.

## Learning Outcomes
- I improved my understanding of `fetch` and how to handle API success and failure states.
- I learned how to combine multiple UI rules using a shared application state object.
- I practiced building stronger client-side form validation instead of relying on only basic HTML input types.
- I improved my workflow for reviewing AI-generated code critically instead of accepting it immediately.

## Responsible Use and Modifications
- I reviewed all AI suggestions before adding them to the project.
- I modified generated code to match the assignment requirements, my own portfolio content, and the project structure.
- I removed ideas that were unnecessary or too complex for this assignment.
- I tested the logic manually and kept only the parts I understood and could explain.
- AI was used as a support tool for productivity and problem-solving, not as a replacement for my own work or understanding.
