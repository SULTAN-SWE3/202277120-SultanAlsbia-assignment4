# AI Usage Report

## Tools Used and Use Cases

### ChatGPT / Codex
- Helped interpret the Assignment 4 requirements and convert them into an implementation checklist.
- Suggested ways to expand the Assignment 3 portfolio into a fuller personal web application.
- Helped review JavaScript logic for filtering, sorting, saved preferences, checklist progress, and form validation.
- Helped improve the README, technical documentation, and presentation script.
- Helped identify areas where error handling, accessibility, and responsive behavior could be improved.

### GitHub Copilot
- Used as a possible code completion assistant while writing repetitive HTML, CSS, and JavaScript patterns.
- Useful for speeding up repeated card structures, form logic, and simple helper functions.

## Benefits
- AI helped turn a long assignment brief into smaller development tasks.
- It helped compare the existing Assignment 3 work against the Assignment 4 rubric.
- It improved productivity when writing repeated UI structures and documentation sections.
- It helped identify edge cases such as failed API requests, empty filter results, invalid form input, and saved browser state.
- It supported the presentation preparation by turning the final feature set into a clear speaking script.

## Challenges and Limitations
- Some suggestions were too broad and needed to be simplified so the project stayed clear and appropriate for a static portfolio.
- AI-generated wording sometimes sounded generic, so it was edited to match my own project and assignment requirements.
- JavaScript suggestions still required manual review to make sure the code worked with the actual HTML element IDs and project structure.
- AI could not replace testing. The final behavior still needed to be checked in the browser and with syntax validation.

## Learning Outcomes
- I practiced using AI as a support tool while still keeping responsibility for the final code.
- I improved my understanding of how to organize front-end state using a shared JavaScript object.
- I learned how to combine filters, search, sorting, and dynamic rendering in one project explorer.
- I improved my use of `localStorage` for theme settings, visitor preferences, and checklist progress.
- I learned how to make API-based UI more reliable by showing loading, success, empty, and error states.
- I improved my documentation workflow by keeping technical details, AI usage, and presentation material separate.

## Responsible Use and Modifications
- I reviewed AI-assisted code before adding it to the project.
- I changed wording, feature names, and data to match my own portfolio and assignment.
- I removed suggestions that were unnecessary, too complex, or not aligned with the rubric.
- I checked that the final code uses the required local folder structure.
- I kept the project simple enough to understand and explain during the presentation.
- I documented AI involvement clearly for academic integrity.

## Examples of Human Review
- The final app keeps a static HTML/CSS/JavaScript structure instead of adding an unnecessary framework.
- The project uses real local assets from `assets/images/` instead of relying only on decorative graphics.
- The contact form validation messages were adjusted to be specific and user-friendly.
- The GitHub API section was kept readable and includes graceful error handling.
- The presentation script was edited to match the implemented features exactly.
